## 完成摘要

P1-02、P1-03、P1-04、P1-05、P1-06、P1-07、P2-01、P2-02、P3-01、P3-02、P3-03 均已完成，并在优化清单中标记状态。

### P1-02 LLM Client

- 删除非 streaming 实现；普通正文、工具续轮、plan、compaction 和 summary 全部使用 SSE。
- 增量组装 `content`、`reasoning_content/reasoning` 和分片 `tool_calls.arguments`。
- 完整 assistant message 结束后才提交 Agent history 和 Worklog；TUI 只打印一次身份前缀并实时追加 chunk。
- 支持 request/connect timeout、context cancellation、Ctrl+C 取消当前请求。
- 408、429、5xx 与建连错误只在尚未收到正文/工具增量时有限重试；已有语义输出后不重放。
- base URL 使用 TrimRight 规范拼接；非 SSE、API error、无 `[DONE]`、损坏 JSON、空 stream 均返回明确错误。
- usage 缺失时估算并在 Context Bar 标记 `estimated`。

### P1-03 统一 Worklog

- 会话目录改为 `worklogs/YYYY-MM-DD/session_<id>/`，包含 `session.md`、`events.jsonl`、`artifacts/`。
- JSONL 事件带 schema、UTC timestamp、单调 sequence、session/request/event/tool_call ID、status、sensitivity 和 redaction status。
- 用户/ELIZA 对话、LLM 生命周期、重试/取消、工具、审批、TUI 命令、mode/role、plan、memory、skill、compaction、checkpoint 和 emergency summary 统一进入一个管理器。
- 每个关键事件写入后 Sync；损坏尾行不影响此前完整事件恢复。
- 大输出进入有上限 artifact，记录相对路径、大小、原始大小、truncated 和 SHA-256。
- API key、Authorization、password、token 等基础模式在日志写入前脱敏；日志不写 ANSI。
- 删除工具各自维护的私有 ResultLog。

### P1-04 Memory

- 正常首次运行创建 `memory/user.md`、`memory/project.md`、`memory/agent.md`，仅缺失时创建，不覆盖已有文件。
- 三类内容带 `UNTRUSTED MEMORY SOURCE` 标签，设置单文件与总注入上限。
- save/forget 展示目标和精确内容，并要求本次 TUI 审批；拒绝后文件不变，批准不跨调用复用。
- `-q` 禁止 memory 修改；autopilot 和 `agent.md` 也不能绕过审批。
- `/memory` 只显示状态、大小和授权边界。

### P1-05 Skill

- 仅使用二进制同目录本地 `skills/`，启动建立轻量索引，`skill_view` 按需加载正文或相对资源。
- 校验目录/文件软链接、frontmatter、名称一致性与唯一性、单文件/索引上限、资源扩展名和目录逃逸。
- 损坏 skill 隔离并显示错误，不阻塞其他 skill。
- 增加 `/skills`、reload、enable、disable；启停不改用户文件。
- Skill 内容带不可信边界，不能提升 Tool Policy、memory 或 workspace 权限。

### P1-06 Role 与 Tool Policy

- Role 同时用于 LLM tool definition 过滤和执行前再次授权。
- writer 不暴露 run_command，write_file 每次审批；security 不暴露 write_file，命令强制 readonly 校验。
- readonly + coder 仍只读；未知 role、伪造 tool call 和 skill 指令不能绕过执行层。
- `/tools` 展示 mode + role 交集后的启用/禁用工具和原因。

### P1-07 Plan

- Plan/Step 使用稳定 ID、状态、时间、attempt 和历史结果。
- 生成 draft；`/execute` 确认后逐步执行，每步前后原子更新 JSON 状态和 Markdown 投影。
- 步骤失败自动 paused，不执行后续步骤；支持 `/retryplan`、`/skipstep`、`/cancelplan`。
- 异常退出的 running plan 在重启时恢复为 paused，running step 标为 failed；已完成步骤不重跑。
- `/new` 对 active plan 提供保留或取消选择。

### P2 CLI 与 doctor

- `-h`、`-help`、`--help` 输出一致；系统终端命令与 TUI 命令严格分区。
- help/version 在 env、目录、锁、API 和 Worklog 初始化之前返回。
- 未知参数非零退出并给出相近参数建议。
- `doctor`/`--doctor` 检查版本、Go 构建、系统、env、模型、脱敏 key、URL、streaming、超时/重试、命令/文件/compaction 策略、锁、shell、路径、memory 与 skill。
- 网络探测按 DNS/TCP/TLS/HTTP 分类；`--offline` 跳过网络。退出码 0/1/2 分别表示通过/警告/阻断失败。
- 对 doctor 前后文件路径、mtime 和大小快照做 diff，结果无变化。

### P3 UI

- 新增集中 Renderer 和深红/白主题常量，不在业务代码散落 ANSI。
- 根据 `docs/logo.png` 以 40×20 Braille 单元重构高精度点阵少女，相当于 80×80 可表达点，并复用项目集中定义的粉发、肤色、制服、电脑与高光色。
- 移除顶部巨型 ELIZA-AGENT 字标，保留原有 `ELIZA-AGENT (DRC Bank ver.)` 与 `Powered By MUY & ELIZA` 文字。
- 110 列及以上在参数框右侧展示少女；40–109 列将少女居中置顶且下方参数框不预留右栏；更窄终端安全省略点阵。
- Banner 标签对齐并按显示宽度换行；API key 只显示脱敏值。
- ELIZA/USER 使用同一圆点字形和不同主题色；无色模式保留文本身份。
- 工具输出使用紧凑 name/status/duration/exit/truncated 块，正常结果走 stdout，诊断走 stderr。
- `--plain`、`--no-color`、`NO_COLOR`、`TERM=dumb`、非 TTY 和 Windows 保守退化均不依赖 ANSI。
- `/help` 只包含 TUI 命令；未知斜杠命令被本地拦截并建议相近命令。
