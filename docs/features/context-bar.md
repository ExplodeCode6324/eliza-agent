# Context Bar（上下文用量条）

> **文件**: `contextbar.go`  
> **版本**: v0.2.1

---

## 设计目标

每次 LLM 回复后显示一行上下文用量条，类似 Hermes 的底部状态栏。

## 显示格式

```
📊 [████████░░] 78%  99.8K / 128.0K tokens | 轮次:3 消息:12
```

| 元素 | 说明 |
|------|------|
| `[████████░░]` | 10 格进度条，颜色编码 |
| `78%` | 上下文填充百分比 |
| `99.8K / 128.0K` | 已用 / 总量 |
| `轮次:3 消息:12` | 统计信息 |

## 颜色编码

| 用量 | 颜色 | 含义 |
|------|------|------|
| < 50% | 绿色 | 安全 |
| 50-80% | 黄色 | 注意 |
| > 80% | 红色 | 危险，即将触发压缩 |

## 数据来源

1. **API 返回**（优先）: `usage.prompt_tokens` — 最后一次 API 调用的实际 prompt token 数
2. **估算**（兜底）: 消息字符总数 / 3.5 — 未调用过 API 时的粗略估计

## 配置

```json
{
  "model": {
    "context_window": 131072   // 模型上下文窗口上限
  }
}
```

常见模型窗口:
- deepseek-chat (v4-flash): 128K (131072)
- deepseek-reasoner: 64K (65536)
- gpt-4o: 128K
- claude-3.5-sonnet: 200K

## 与压缩联动

`compress.go` 现在优先按百分比触发（默认 >75%），消息数（>30条）作为兜底：

```
context 填充 > 75% → 自动触发压缩
或
消息数 > 30 条 → 兜底触发
```
